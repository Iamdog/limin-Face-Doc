# nextTick与异步更新队列面试题

## 知识点

- Vue 的 DOM 更新是异步批处理，不会在每次状态变更后立刻同步刷新 DOM。
- 同一事件循环内的多次状态修改会被合并，减少重复渲染。
- `nextTick` 用于等待本轮 DOM 更新完成后再执行逻辑，常用于读取最新 DOM 状态。

## 实战场景

```ts
count.value++
await nextTick()
console.log(el.value?.offsetHeight)
```

- 更新数据后需要立刻测量高度、滚动位置、焦点等 DOM 信息时使用。
- 不应滥用 `nextTick`，优先通过数据驱动解决时序问题。

## 高频面试题

### 1. 为什么 Vue 要做异步更新？
答：为了合并同一轮中的多次修改，减少无效渲染，提高性能。

### 2. `nextTick` 是不是“延时函数”？
答：不是。它语义是等待 Vue 完成本轮 DOM 刷新，而不是固定时间延迟。

### 3. 什么时候不需要 `nextTick`？
答：当逻辑只依赖响应式数据、不依赖已更新 DOM 时通常不需要。
