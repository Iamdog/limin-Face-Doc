# 组件通信与面试题

## 知识点

- 父传子：`props`。
- 子传父：`emit`。
- 跨层传递：`provide / inject`。
- 非父子共享状态：Pinia（推荐）或事件总线（谨慎）。

## 工程建议

- `props` 只读，避免子组件直接修改。
- `emit` 事件名语义化，例如 `update:page`、`submit`。
- `provide/inject` 适合主题、表单上下文等“弱耦合共享”。

## 高频面试题

### 1. 为什么不推荐事件总线作为主通信手段？
答：可追踪性差、调试困难、维护成本高，规模变大后问题明显。

### 2. `v-model` 本质是什么通信模式？
答：`prop + emit` 语法糖，默认是 `modelValue` 和 `update:modelValue`。

### 3. 多层组件如何避免“props drilling”？
答：可用 `provide/inject` 或将状态上收至 Pinia。
