# 响应式原理与面试题

## 知识点

- Vue3 基于 `Proxy` 实现响应式追踪。
- 读取数据时会“收集依赖”（track），修改数据时会“触发更新”（trigger）。
- `ref` 适合基础类型或单值状态；`reactive` 适合对象结构。
- `computed` 有缓存，依赖不变时不会重复执行。
- `watch` 用于副作用；`watchEffect` 自动收集依赖。

## 常见坑

- 结构赋值会丢失响应式引用语义。
- `reactive` 直接整体替换对象可能导致引用链中断，推荐按字段更新。
- 深层 `watch` 成本高，尽量缩小监听范围。

## 高频面试题

### 1. `ref` 和 `reactive` 如何选择？
答：单值或基础类型优先 `ref`；对象状态可用 `reactive`，但复杂场景也可用多个 `ref` 拆分。

### 2. `computed` 和 `watch` 的区别？
答：`computed` 产出派生状态并带缓存；`watch` 处理异步请求、日志上报等副作用。

### 3. Vue2 和 Vue3 响应式核心差异？
答：Vue2 主要依赖 `Object.defineProperty`，Vue3 使用 `Proxy`，对新增/删除属性、数组场景支持更完整。
