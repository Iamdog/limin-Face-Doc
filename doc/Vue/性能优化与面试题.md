# 性能优化与面试题

## 知识点

- 首屏优化：路由懒加载、组件按需加载、资源压缩。
- 渲染优化：减少不必要响应式、合理拆分组件、避免大对象深监听。
- 列表优化：稳定 `key`、虚拟列表、分页/分片渲染。
- 工程优化：构建分析、缓存策略、CDN 与静态资源版本化。

## 诊断路径

- 先定位瓶颈（网络、脚本、渲染、内存）。
- 再做针对性优化并复测，避免“拍脑袋优化”。
- 建议建立性能基线指标（FCP、LCP、交互耗时）。

## 高频面试题

### 1. Vue 项目首屏慢如何排查？
答：从包体积、请求数量、关键资源加载顺序入手，配合构建分析工具定位。

### 2. 为什么 `key` 对列表渲染重要？
答：它帮助虚拟 DOM 正确复用节点，减少错误复用和额外更新成本。

### 3. `watch` 用多了会有什么问题？
答：会增加依赖追踪与执行开销，复杂场景可能导致性能下降和维护困难。
