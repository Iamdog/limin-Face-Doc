# 路由权限守卫与面试题

## 知识点

- 权限控制通常放在全局前置守卫 `beforeEach`。
- 常见流程：未登录拦截 -> 登录页 -> 登录成功回跳。
- 结合路由元信息 `meta` 可表达鉴权规则（如 `requiresAuth`、`roles`）。

## 参考实现

```ts
router.beforeEach((to) => {
  const token = userStore.token
  if (to.meta.requiresAuth && !token) {
    return { path: '/login', query: { redirect: to.fullPath } }
  }
  if (to.meta.roles && !to.meta.roles.includes(userStore.role)) {
    return { path: '/403' }
  }
})
```

## 高频面试题

### 1. 前端权限控制能保证安全吗？
答：不能。前端权限主要用于体验和流程控制，真正安全依赖后端鉴权。

### 2. 为什么推荐基于 `meta` 做权限声明？
答：路由与权限规则集中管理，可读性和可维护性更好。

### 3. 守卫里做异步请求有什么风险？
答：会影响路由切换时延，需要处理超时、失败回退和并发跳转。
