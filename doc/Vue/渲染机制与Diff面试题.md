# 渲染机制与Diff面试题

## 知识点

- Vue 通过虚拟 DOM 描述 UI，再将变更映射到真实 DOM。
- 更新时会进行 Diff，尽量复用已有节点，最小化 DOM 操作。
- 列表更新依赖稳定 `key`，否则会导致错误复用和额外重排。
- 编译阶段的静态提升、Patch Flag 等优化可减少运行时开销。

## 常见优化点

- 列表项必须提供稳定主键，不使用随机值或索引（除非静态列表）。
- 大组件拆分成更小渲染单元，避免一处状态触发大范围更新。
- 对不会变化的大块内容可考虑 `v-once` 或缓存策略。

## 高频面试题

### 1. Vue 的 Diff 是“全量比较”吗？
答：不是。它会基于节点类型、层级和 `key` 做有策略的比较与复用。

### 2. 为什么不建议用索引当 `key`？
答：插入/删除时索引会漂移，可能导致节点错误复用，出现状态错乱。

### 3. 编译优化和运行时优化分别做什么？
答：编译优化减少运行时需要判断的内容；运行时优化负责高效执行更新。
