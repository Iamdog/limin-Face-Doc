# 生命周期与面试题

## 知识点

- 初始化阶段：`setup`。
- 挂载阶段：`onBeforeMount`、`onMounted`。
- 更新阶段：`onBeforeUpdate`、`onUpdated`。
- 卸载阶段：`onBeforeUnmount`、`onUnmounted`。
- 缓存组件：`onActivated`、`onDeactivated`。

## 使用建议

- DOM 依赖逻辑放 `onMounted`。
- 事件订阅、定时器、外部实例要在 `onUnmounted` 清理。
- 避免在 `onUpdated` 无条件改响应式状态，否则可能触发循环更新。

## 高频面试题

### 1. `created` 在 Vue3 去哪了？
答：在 Composition API 中通常用 `setup` 替代其初始化职责。

### 2. 请求接口放在哪个生命周期？
答：通常放在 `onMounted` 或 `setup` 触发的业务方法中，取决于是否依赖 DOM。

### 3. `keep-alive` 场景为什么要用 `onActivated`？
答：组件不会被销毁重建，重新激活时要在 `onActivated` 刷新数据或恢复状态。
