# Android 动画体系

动画体系是 Android 常见考点，通常会问属性动画、补间动画、帧动画区别，以及为什么属性动画更灵活。

## 1. Android 动画主要分哪几类？

### 1.1 帧动画

- 按顺序播放一组图片资源。
- 本质像逐帧切换。
- 实现简单，但资源占用可能较高。

### 1.2 补间动画

- 作用于 View 的显示效果，比如平移、缩放、透明度、旋转。
- 视觉上动了，但 View 的真实属性值未必改变。

### 1.3 属性动画

- 直接修改对象属性值。
- 不只适用于 View，也可作用于任意对象。
- 现代开发里最常用。

## 2. 为什么属性动画更推荐？

- 真正修改属性值。
- 可组合、可监听、可扩展。
- 不局限于 View。

## 3. 常见属性动画 API

- `ObjectAnimator`
- `ValueAnimator`
- `AnimatorSet`

示例：

```kotlin
ObjectAnimator.ofFloat(view, "alpha", 1f, 0.5f, 1f).apply {
    duration = 300
    start()
}
```

## 4. `ValueAnimator` 和 `ObjectAnimator` 的区别

- `ValueAnimator` 只负责产生数值变化。
- `ObjectAnimator` 在 `ValueAnimator` 基础上，自动把值设置到目标属性。

## 5. 插值器和估值器

### 插值器 Interpolator

- 决定动画执行节奏。
- 比如匀速、加速、减速、回弹。

### 估值器 TypeEvaluator

- 决定起始值到结束值之间如何计算中间值。

## 6. 为什么动画有时会引起卡顿？

- 主线程负担重。
- 频繁触发重绘。
- 动画期间做了复杂布局或图片处理。
- 低端机 GPU/CPU 压力大。

## 7. View 动画和属性动画的经典区别

- View 动画很多时候只是“看起来在动”。
- 属性动画是对象真实值在变化。
- 所以点击区域、布局位置等行为在两者下可能不一样。

## 8. 常见面试题

### Q1: 补间动画和属性动画有什么区别？
A: 补间动画更多是视觉变换，不一定真的改变对象属性；属性动画会真实修改属性值，扩展性更强。

### Q2: 为什么按钮做了平移动画后，点击区域有时还是原来的位置？
A: 这是典型补间动画现象，视觉位置变了，但真实属性和布局位置没变。

### Q3: `ValueAnimator` 能作用于普通对象吗？
A: 可以。它本质只生成数值，开发者可以自己把数值应用到任何对象上。

### Q4: 动画为什么会造成内存泄漏？
A: 如果 Animator 长时间持有 View、Activity 引用，又没有及时 `cancel()`，页面销毁后对象可能无法及时回收。

### Q5: 如何减少动画卡顿？
A: 降低过度绘制、减少布局层级、避免动画中做重逻辑、尽量让动画更多交给渲染管线处理。

## 9. 小结

动画题面试重点很固定：类型、区别、底层影响和性能。把“补间动画只改表现，属性动画改真实值”讲清楚，基本就能过。
