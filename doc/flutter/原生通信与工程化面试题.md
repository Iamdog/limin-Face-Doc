# Flutter原生通信与工程化面试题

这部分是中高级面试高频：Flutter 与原生如何协作、插件如何落地、工程如何构建发布。

## 1. Flutter 与原生通信方式

- `MethodChannel`：请求-响应模型，最常用。
- `EventChannel`：事件流场景，如传感器、定位。
- `BasicMessageChannel`：自定义双向消息协议。

## 2. 插件机制

- Dart 层对外暴露统一 API。
- iOS/Android 分别实现平台代码。
- 通过平台通道桥接，屏蔽平台差异。

## 3. 常见通信问题

- 通道名冲突导致调用失败。
- 参数序列化不一致导致类型错误。
- 生命周期不一致导致回调丢失或空引用。

## 4. 工程化关键点

- 多环境配置（dev/staging/prod）。
- 资源与包体积治理（图片、字体、依赖裁剪）。
- CI/CD 自动化构建与签名发布。

## 5. 混合工程实践

- 增量接入：在原生项目中嵌入 Flutter 页面。
- 路由和账号体系统一。
- 性能敏感或平台强依赖模块可保留原生实现。

## 6. 高频面试题

### Q1: `MethodChannel` 和 `EventChannel` 怎么选？
A: 一次性调用选 `MethodChannel`，持续事件订阅选 `EventChannel`。

### Q2: Flutter 插件为什么要分平台实现？
A: 因为硬件能力和系统 API 本质上仍由各平台原生提供。

### Q3: 混合开发里最大风险是什么？
A: 路由/生命周期/状态同步不一致，导致页面管理和问题排查成本上升。

### Q4: 如何降低 Flutter 包体积？
A: 精简依赖、压缩资源、按需加载、关闭无用 ABI/资源。

## 7. 小结

把通道模型、插件结构、混合工程边界讲清楚，原生协作类问题就能答得很完整。
