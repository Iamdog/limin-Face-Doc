# Flutter性能优化与面试题

Flutter 性能题建议按“构建、布局绘制、图片内存、异步任务”四条线回答。

## 1. 构建层优化

- 尽量使用 `const` 构造，减少重复创建。
- 拆分 Widget，缩小重建范围。
- 避免在 `build` 中执行重计算。

## 2. 布局与绘制优化

- 降低过深组件树层级。
- 高频重绘区域使用 `RepaintBoundary`。
- 慎用 `Opacity`、`Clip` 等可能增加开销的组件。

## 3. 列表优化

- 使用 `ListView.builder` 按需构建。
- 列表项尽量保持轻量、可复用。
- 避免在滑动过程中触发大量 setState。

## 4. 图片与内存

- 控制图片分辨率与解码尺寸。
- 使用缓存策略，避免重复下载/解码。
- 警惕大图、GIF、长图造成内存峰值。

## 5. 监控与定位

- DevTools 查看 UI/GPU 帧时间。
- 用 Timeline 分析卡顿阶段。
- 关注 16ms 帧预算与掉帧比例。

## 6. 高频面试题

### Q1: 为什么 `const` 能优化性能？
A: 常量对象可复用，减少对象分配与比较成本，降低 GC 压力。

### Q2: `RepaintBoundary` 什么时候有收益？
A: 父节点频繁刷新但子区域内容稳定时，隔离重绘可显著降低绘制成本。

### Q3: Flutter 卡顿排查第一步做什么？
A: 先确认是构建慢、布局绘制慢，还是图片解码/IO 导致，再针对性优化。

### Q4: Isolate 在性能优化里有什么作用？
A: 将重 CPU 任务移到后台 Isolate，避免阻塞主 Isolate 导致掉帧。

## 7. 小结

性能优化不是单点技巧，而是“先定位瓶颈，再做针对性优化，再复测验证”的闭环。
