# Flutter布局机制与面试题

Flutter 布局题的核心是约束模型。只要能解释清楚约束传递，80% 布局问题都能回答。

## 1. 约束模型

- 父组件给子组件约束（min/max 宽高）。
- 子组件在约束内计算尺寸。
- 父组件根据子组件尺寸决定最终摆放。

## 2. 常见布局组件定位

- `Row` / `Column`：线性布局。
- `Stack`：层叠布局。
- `Expanded` / `Flexible`：弹性分配剩余空间。
- `Wrap`：自动换行流式布局。
- `LayoutBuilder`：按父约束动态构建。

## 3. 溢出问题怎么答

- `RenderFlex overflowed` 本质是子组件总尺寸超出父约束。
- 常见修复：用 `Expanded/Flexible`、限制宽高、改可滚动容器。

## 4. 长列表与复杂页面

- 长列表优先 `ListView.builder` / `SliverList`。
- 复杂页面可用 `CustomScrollView + Sliver` 统一滚动模型。
- 避免在列表中使用昂贵同步计算。

## 5. 响应式适配

- 使用 `MediaQuery` 获取屏幕信息。
- 使用断点策略处理手机/平板。
- 字体与间距适配要跟设计规范绑定。

## 6. 高频面试题

### Q1: `Expanded` 和 `Flexible` 有什么区别？
A: `Expanded` 等价于 `Flexible(fit: FlexFit.tight)`，必须占满分配空间；`Flexible` 可选择紧/松约束。

### Q2: 为什么会出现无限高度错误？
A: 通常是可滚动组件嵌套时约束不明确，子组件拿到 unbounded 约束导致布局失败。

### Q3: 什么时候用 Sliver？
A: 多段异构滚动布局、吸顶头部、瀑布混排等复杂场景优先考虑 Sliver。

## 7. 小结

面试时优先讲“约束链路 + 典型错误 + 修复策略”，比背组件 API 更有说服力。
